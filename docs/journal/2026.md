## 2026-06-05 — Session Summary

**Decisions:**

- Built custom ADR skill (MADR format) over installing adr-kit — adr-kit is designed for team-scale enforcement (pre-commit blocking, LLM judge on every commit, pattern/import rules). Our decisions include workflow and process decisions that adr-kit's enforcement engine can't capture. Right-sized for solo projects; revisit adr-kit at 5+ active developers or when compliance requirements emerge.
- Chose MADR format over Nygard minimal for ADRs — MADR adds "Considered Options" and "Decision Drivers" sections, which capture what was rejected and why. This directly prevents re-litigating the same decisions in future sessions.

- adr-kit deferred rather than discarded — noted in global CLAUDE.md New Project Checklist as something to assess at project start, alongside RFCs, AGENTS.md, and log4brains. Worth adopting at enterprise scale; not needed now.
- Per-year journal files (`docs/journal/YYYY.md`) over per-session files — avoids clutter while keeping entries scannable; a new file each year keeps history manageable.
